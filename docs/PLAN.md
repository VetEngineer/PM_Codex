# Planning Process (Codex)

Codex produces implementation-ready plans without making code changes.

## Inputs

- Requirements or tickets
- Constraints (tech stack, dependencies, deadlines)
- Design specs from gemini-cli

## Output Structure

1. Objective
2. Scope (in / out)
3. Assumptions
4. Detailed steps (ordered)
5. Files to touch
6. Risks and mitigations
7. Acceptance criteria
8. Test plan
